COMPRESSION RULES
═══════════════════════════════════════════════════════════════

LANGUAGE:
- Write in telegraphic style. Drop articles (a, the) where meaning is clear.
- Use → for causal chains. Use | for separating fields. Use : for definitions.
- Use ≥ ≤ ~ ≈ instead of "greater than", "approximately", etc.
- Abbreviations are fine if unambiguous in context.

WHAT ALWAYS GETS KEPT (never cut these):
- Any specific number, percentage, threshold, or timeframe
- Any named cognitive bias or failure mode
- Any named model with an equation or structured rule
- Any empirical finding with a cited source
- Any causal mechanism that goes A → B → C
- Any direct quote that is irreplaceable in its original form

WHAT ALWAYS GETS CUT (never keep these):
- Motivational or inspirational framing with no operational content
- Historical background that leads nowhere actionable
- Definitions of terms that any informed reader already knows
- Repetition of the same point in different words
- Academic hedging language ("it could be argued", "some suggest")
- Course logistics, platform instructions, grading information
- Anything describing what the next section will cover

SPECIAL CASES:
- If the file is personal notes/journal: preserve raw phrasing, do not interpret
- If the file is a course transcript: extract only insight + mechanism, cut pedagogy
- If the file is a research paper: keep findings + numbers, cut literature review
- If the file is a book summary: keep frameworks + key examples, cut narrative
- If the file is news/analysis: keep event + mechanism + affected assets, cut opinion
